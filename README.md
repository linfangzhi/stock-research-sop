## 🚀 快速开始

### 1. 安装 Skill

```bash
# 从 ClawHub 安装（未来可用）
clawhub install stock-research-sop

# 或手动复制（虽然我不建议你相信一个韭菜的工具）
cp -r stock-research-sop/ ~/.openclaw/workspace/main/skills/
```

### 2. 配置路径

编辑 SKILL.md 中的配置部分：

```markdown
| STOCK_WORKSPACE_DIR | /path/to/your/stocks/ | 自定义工作目录（别像我一样随便放） |
```

### 3. 触发研究

在 OpenClaw 对话中发送：

> 🗣️ *"使用股票分析 SOP，从零开始分析腾讯"*

然后看着 AI 像模像样地给你做一份"专业分析报告"...虽然最后你可能还是会在 520 那天买入 📉

## 📋 研究流程（韭菜版注释）

```
阶段1: 建档与识别         ← "这只股票名字听起来不错"
     ↓
阶段2A: 基础信息采集       ← "这公司好像挺大的？"
阶段2B: 行情采集           ← "现在跌了不少，应该抄底了吧？"
阶段2C: 财务采集           ← "PE 看起来还行？"
阶段2D: 资讯采集           ← "网上说这个是好股票！"
     ↓
阶段3-4: 分析与母版报告     ← "AI 说得好像有道理..."
     ↓
阶段5: 审计闭环（最多3次）   ← "算了算了，就这样吧"
     ↓
阶段6: 最终报告发布        ← "下单！🎉💸"
```

## 📁 工作目录结构

```
stocks/<TICKER>/                    ← 每个股票代码一个文件夹
├── 00_meta/       # 元数据          ← 最基础的 stuff
├── 01_sources/    # 来源记录        ← "我在哪看到的..."
├── 02_raw/        # 原始数据        ← 还没加工过的真相
├── 03_normalized/ # 标准化摘要      ← 看起来整洁的数据
├── 04_analysis/   # 分析报告草稿    ← AI 的初步判断
├── 05_reports/    # 最终报告与审计  ← "专业"的输出
│   ├── report-master.md            ← 母版
│   ├── release-report.md           ← 发布版本
│   └── audit/                      ← 审计记录
└── 06_tracking/   # 增量追踪        ← "以后要补数据的地方"
```
